# Cao - Product Concept

## Product Vision

**「理想の自分を、言葉で伝える」を可能にする**

美容医療やエステにおいて、顧客が抱える最大の課題は「自分の理想を専門家に正確に伝えられない」こと。Caoは、AIによる顔分析とビジュアルシミュレーションを通じて、顧客と施術者の間のコミュニケーションギャップを解消します。

---

## Target Users

### Primary: 美容医療検討者

**ペルソナ**
- 年齢: 25-45歳 女性
- 美容医療（ヒアルロン酸注入、ボトックス、糸リフトなど）を検討中
- SNS（Instagram、TikTok）で美容情報を日常的に収集
- 可処分所得があり、自己投資に積極的
- 「失敗したくない」「自然に変わりたい」という思いが強い

**行動特性**
- 施術前に複数のクリニックを比較検討
- 症例写真やレビューを徹底的にリサーチ
- カウンセリングに行っても即決できない
- 友人や家族には相談しづらいと感じている

---

## User's Job to Be Done

### Core Job
**「自分が美容医療でどう変われるかを、施術前に具体的にイメージしたい」**

### Related Jobs
1. 自分の顔の特徴を客観的に理解したい
2. 「理想の顔」を明確にしたい
3. 施術者に自分の希望を正確に伝えたい
4. 施術後の仕上がりを事前に確認したい
5. どの施術が自分に必要かを判断したい

---

## User's Pains（ペイン）

### 1. 言語化できない
> 「もう少し目を大きく...でも不自然にならない程度に...」

- 自分の理想を言葉で説明するのが難しい
- 「なんとなくこう」というイメージはあるが、具体化できない
- 施術者に伝えるときに曖昧な表現になってしまう

### 2. イメージが湧かない
> 「この施術を受けたら、私の顔はどうなるの？」

- 症例写真は他人の顔なので参考にならない
- 自分の顔がどう変わるかが想像できない
- 「やりすぎ」になるのが怖い

### 3. 比較できない
> 「芸能人の〇〇みたいな鼻にしたい」と言っても...

- 理想の顔と自分の顔を並べて比較できない
- どの程度近づけるのかがわからない
- どのパーツを変えれば理想に近づくかわからない

### 4. 共有できない
> 「先生に私の希望、ちゃんと伝わってるかな...」

- カウンセリングで施術者と認識がズレることがある
- 自分のイメージを視覚的に共有する手段がない
- 施術後に「思っていたのと違う」となるリスク

### 5. 決断できない
> 「本当にこれでいいのかな...」

- 不可逆な施術への不安
- 高額な投資への躊躇
- 情報が多すぎて判断軸がない

---

## Solution

### Core Solution: Face Morphing Simulation

ユーザーの現在の顔写真と理想の顔を入力し、**段階的なモーフィングシミュレーション**を生成。
```
[現在の顔] → [25%] → [50%] → [75%] → [理想の顔]
```

**「私は50%くらいまで近づきたい」**を視覚的に確認・共有できる。

### Solution Features

| Feature | Description |
|---------|-------------|
| 段階的シミュレーション | 0%〜100%の変化を段階的に可視化 |
| 髪型・背景の保持 | 顔パーツのみ変化、自然な仕上がり |
| パーツ別調整 | 目、鼻、口、輪郭を個別に調整可能 |
| 結果の共有 | URLで施術者にシミュレーションを共有 |
| GIFアニメーション | 変化の過程を動画で確認 |

---

## How Solution Resolves Pains

| Pain | Solution | Resolution |
|------|----------|------------|
| **言語化できない** | ビジュアルシミュレーション | 「この画像の状態にしたい」と視覚的に伝えられる |
| **イメージが湧かない** | 自分の顔でシミュレーション | 他人の症例ではなく、自分の顔の変化を確認できる |
| **比較できない** | 段階的モーフィング | 現在→理想を並べて、どこまで近づくか比較できる |
| **共有できない** | URL共有機能 | 施術者とシミュレーション結果を共有し、認識を合わせられる |
| **決断できない** | パーツ別・段階別調整 | 「まずは鼻だけ25%」など、小さく始める判断ができる |

## Current Alternatives（代替手段）

### 1. 症例写真を見せる
**方法**: クリニックの症例写真やSNSの投稿を施術者に見せる

**課題**:
- 他人の顔なので「私の場合はどうなるか」がわからない
- 骨格や肌質が異なるので同じ結果にならない
- 「この人みたいに」と言っても解釈がズレる

### 2. 芸能人の写真を見せる
**方法**: 「〇〇さんの鼻にしてください」と参考画像を提示

**課題**:
- 現実的でない期待を持ってしまう
- 施術者が困惑する（「無理です」と言いづらい）
- パーツ単位の指定が難しい

### 3. 加工アプリで自撮りを編集
**方法**: Snow、FaceAppなどで理想の顔を作る

**課題**:
- 過度な加工になりがち（現実的でない）
- 美容医療のシミュレーションとして設計されていない
- 髪型や背景も変わってしまい、変化がわかりづらい
- 施術者に見せても「これは無理」となる

### 4. 施術者の経験に委ねる
**方法**: 「お任せします」「自然な感じで」と伝える

**課題**:
- 施術者の解釈と自分の期待がズレるリスク
- 後から「思っていたのと違う」となりやすい
- 主体的な意思決定ができない

### 5. カウンセリングで相談
**方法**: 施術者と対話しながらイメージを固める

**課題**:
- 言語だけでは伝わらない
- カウンセリング時間の制約
- 複数クリニックで同じ説明を繰り返す負担

---

## Why Cao Wins

| 比較軸 | 代替手段 | Cao |
|--------|----------|-----|
| 自分の顔で確認 | ❌ 他人の顔 | ✅ 自分の顔 |
| 段階的な変化 | ❌ Before/Afterのみ | ✅ 0%〜100%段階的 |
| パーツ別調整 | ❌ 全体のみ | ✅ パーツごとに調整 |
| 現実的な仕上がり | ❌ 過度な加工 | ✅ 髪型・背景保持で自然 |
| 施術者との共有 | ❌ 言語のみ | ✅ URL共有で視覚的に |
| 美容医療に特化 | ❌ 汎用アプリ | ✅ 施術シミュレーション目的 |

## User Story Examples

### Story 1: 初めての美容医療
> 「ヒアルロン酸で涙袋を作りたいけど、やりすぎたらどうしよう...」

1. Caoで自分の顔をアップロード
2. 「涙袋あり」の理想画像を設定
3. 25%、50%、75%のシミュレーションを確認
4. 「50%くらいが自然でいいな」と判断
5. シミュレーション画像を持ってカウンセリングへ
6. 「この画像くらいにしてください」と具体的に伝える

### Story 2: なりたい顔がある
> 「この韓国アイドルみたいな顔になりたい！」

1. 自分の顔と理想の顔（アイドル）をCaoに入力
2. モーフィングで「どこがどう違うか」を可視化
3. 「鼻筋と目の大きさが違うんだ」と客観的に理解
4. パーツ別に調整し、現実的な目標を設定
5. 「まず鼻から」と優先順位を決められる

### Story 3: 施術者とのすり合わせ
> 「先生と私のイメージ、合ってるかな...」

1. カウンセリング前にCaoでシミュレーション作成
2. 共有URLを施術者に送付
3. 施術者が事前に確認し、カウンセリングで「この状態ですね」と確認
4. 認識のズレがあれば、その場で調整
5. 「このシミュレーションを目標に施術します」と合意

## Key Metrics for Success

### User Value Metrics
- シミュレーション後のカウンセリング予約率
- 施術者への共有率
- リピート利用率
- 「イメージ通りだった」と回答した施術後満足度

### Business Metrics
- MAU（Monthly Active Users）
- シミュレーション生成数
- 有料プラン転換率
- B2Bクリニック導入数
